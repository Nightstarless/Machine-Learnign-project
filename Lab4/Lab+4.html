1. Preparation and Overview¶

1.1 Business understanding¶

In this lab, The dataset is related the Portuguese "Vinho Verde" red wine. The inputs include objective tests (e.g. PH values) and the output is based on sensory data(median of at least 3 evaluations made by wine experts). Each expert graded the wine quality between 0 (very bad) and 10 (very excellent). We know that many points of view are needed to be considered to describe the quality of red wines. When winery produced batches of red wines and wanted to put them into market, how to set the price of these red wine becomes a problem, despite the fact that they may have a traditional way. Also, When customers want to buy red wines, how to buy good quality of red wines is puzzling.

Therefore, finding a quick and effective way to judge the quality of red wine is meaningful. We will use physicochemical properties to explore a effective method to describe the quality of red wines. We will use F1-score to evaluate the model, and the results can serve as a reference in the field of red wine. Both winery and customers will be interested in this results. Because it can not only help the winery set the price scientifically, but also help customers buy best red wines with limited money.

The classification task is to make a multi-layer perceptron, and we will be using the Redwine Dataset to compare the performance of multi-layer perceptrons programmed in scikit-learn and via our own implementation. we have chosen to tackle a classification problem of redwine quality.

This dataset has 12 variables and all variables are numerical. There are 1599 rows of instances and there is no missing data in this dataset.

After doing those processes, the data can be used for classification. When we get the results of classification, we will compute the F1-score to verify the performance of this model.

2 Evaluation¶

2.1 Evaluation Metrics¶

In [315]:

Genarally, We use accuracy, precision, recall, f1-score, ROC-AUC to evalate the model in binary classification problems. For multi-classification problems, Precision, Recall, and F1 can be calculated separately for each class, and integrated as a model evaluation index.

Accuracy: It is the most common and basic evaluation metric.

Precision: It is the proportion of "true" samples judged by all systems that is truly true, that is, TP/(TP+FP).

Recall: It is the proportion of “true” that is judged to be TP/(TP+FN) in all samples that are truly true.

F1-score: The metric designed to comprehensively consider the precision and recall.

ROC-AUC: It is the curve of TP vs FP.

But the objective in different situation is totally diferent, which means that the quality of metrics depends on the goals of the classifier. For instance, email filtering spam, you want it to be more comprehensive (detect all the junk, but there is a lot of useful information is also judged as junk), or you want it to be as accurate as possible (do not always detect useful email as junk)? Obviously, we think that False Positive's damage is greater than False Negative: the loss of important mail if it is sentenced to junk is far greater than the amount of spam in the inbox. However, for the prediction of the earthquake, we hope that recall is very high, that is, we want to predict each earthquake. At this time we can sacrifice percision.

In this lab, our dataset is about the qaulity of red wine and this is an multi-classification question, so some of metrics seems not very meamingful for us and we finally decided to use F1-score to evaluate our model. The reasons are as follows.

First, the quality distribution of red wine isn't relatively balanced, and we know that if the classes are not balanced,the accuracy evaluation has no reference value. Second, We are facing a multi-classification problem, and there are micro or macro average F1-scores that are designed in particular, so we can directly use these. Therefore, in this case, F1-score would tell us that this is actually a pretty good classifier or not.

As for our dataset, we must decide whether a micro or macro average of our scores will be more beneficial.

The macro average is to first calculate the index value for each class and then calculate the arithmetic average of all classes.

The micro average is to establish a global confusion matrix for each instance of the data set without any class, and then calculate the corresponding index.

The difference is that macro average gives each class the same weight, whereas micro average gives each sample the same weight. In the micro average assessment index, the class with more samples dominates the class with fewer samples. However, the quality distribution of red wine isn't relatively balanced and the objective in our dataset is that each class can be accurate enough, because every class is important equally. Therefore, marco average is the appropriate evaluation metric.

In conclusion, we will be using marco average F1-scroe to evaluate the performance of multi-layer perceptrons programmed in scikit-learn and via our own custom implementation.

2.2 Train/Test Validation Methodology¶

In addition to deciding on an evaluation metric that is more appropriate for our business use case, we must also determine the best method by which to divide our data into training and testing splits.

Once again, we are haunted by the curse of unbalanced classes. As far as possible to ensure that each classification of data in the test set and training the same proportion, we choose to use the method of sklearn to split the data.

First of all, we split our data into training and testing splits by 80%/20% in 1.2, and the process of splits are stratified. The testing set will be used for the final customized model and sklearn model to compare the performance.

Then, in the cases of extreme class imbalance, we would want to use a stratified 5-fold cross validation to help ensure a proper ratio between the classes are represented within each fold. We use this stratified 5-fold cross validation set in 3.2 to find the proper hyperparameter for our customized model.

import numpy as np
from scipy.special import expit
import sys

# start with a simple base classifier, which can't be fit or predicted
# it only has internal classes to be used by classes that will subclass it
class TwoLayerPerceptronBase(object):
    def __init__(self, n_hidden=30,
                 C=0, epochs=500, eta=0.001, random_state=None):
        np.random.seed(random_state)
        self.n_hidden = n_hidden
        self.l2_C = C
        self.epochs = epochs
        self.eta = eta
        
    @staticmethod
    def _encode_labels(y):
        """Encode labels into one-hot representation"""
        onehot = pd.get_dummies(y).values.T
            
        return onehot

    def _initialize_weights(self):
        """Initialize weights with small random numbers."""
        W1_num_elems = (self.n_features_ + 1)*self.n_hidden
        W1 = np.random.uniform(-1.0, 1.0, size=W1_num_elems)
        W1 = W1.reshape(self.n_hidden, self.n_features_ + 1) # reshape to be W
        
        W2_num_elems = (self.n_hidden + 1)*self.n_output_
        W2 = np.random.uniform(-1.0, 1.0, size=W2_num_elems)
        W2 = W2.reshape(self.n_output_, self.n_hidden + 1)
        return W1, W2
    
   
    def _activation(self, z):
        if self.phi == 'sigmoid':
            return expit(z)
        elif self.phi == 'linear':
            return z
        elif self.phi == 'relu':
            return z * (z > 0)
        elif self.phi == 'silu':
            return z * expit(z)    
        
    @staticmethod
    def _add_bias_unit(X, how='column'):
        if how == 'column':
            ones = np.ones((X.shape[0], 1))
            X_new = np.hstack((ones, X))
        elif how == 'row':
            ones = np.ones((1, X.shape[1]))
            X_new = np.vstack((ones, X))
        return X_new
    
    @staticmethod
    def _L2_reg(lambda_, W1, W2):
        """Compute L2-regularization cost"""
        # only compute for non-bias terms
        return (lambda_/2.0) * np.sqrt(np.mean(W1[:, 1:] ** 2) + np.mean(W2[:, 1:] ** 2))
    
    def _cost(self,A3,Y_enc,W1,W2):
        '''Get the objective function value'''
        cost = np.mean((Y_enc-A3)**2)
        L2_term = self._L2_reg(self.l2_C, W1, W2)
        return cost + L2_term

class CustizedPerceptron(TwoLayerPerceptronBase):

    
    def __init__(self, nl_hidden=1, phi='relu',cost='cross-entropy',**kwds):
        self.nl_hidden = nl_hidden
        self.n_layers = self.nl_hidden + 1

        self.phi = phi
        self.cost = cost     
 
        super().__init__(**kwds)
            
    def _phi_function(self, A):
        if self.phi == 'sigmoid':
            return A * (1 - A)
        elif self.phi == 'linear':
            return np.ones(A.shape)
        elif self.phi == 'relu':
            return np.ones(A.shape) * (A > 0)
        elif self.phi == 'silu':
            return expit(A) + A * expit(A) * (1 - expit(A))
       
            
            
    def _cost_grad(self, A, Y):
        if self.cost == 'mse':
            return -2 * (Y - A) * (1 - A)
        elif self.cost == 'cross-entropy':
            return (A - Y)

            
    def _get_gradient(self, results, outputs, Y):
        sensitivity = self.network_.copy()
        sensitivity[-1] = self._cost_grad(results[-1], Y)
        self.grads_[-1] = sensitivity[-1] @ results[-2].T
        self.grads_[-1] += self.network_[-1] * self.l2_C
        for i in range(-2, -self.n_layers, -1):
            if i == -2:
                sensitivity[i] = (self._phi_function(results[i]) * 
                                  (self.network_[i+1].T @ sensitivity[i+1]))
            else:
                sensitivity[i] = (self._phi_function(results[i]) * 
                                  (self.network_[i+1].T @ sensitivity[i+1][1:,:]))
                
            self.grads_[i] = sensitivity[i][1:,:] @ results[i-1].T
            self.grads_[i][:,1:] += self.network_[i][:,1:] * self.l2_C  
            
    def _feedforward(self, X):
        result = []
        outputs = []
        
        A = self._add_bias_unit(X, how='column').T
        result.append(A)
        for i in range(self.n_layers - 1):
            Z = self.network_[i] @ A
            A = self._add_bias_unit(self._activation(Z), how='row')
            outputs.append(Z)
            result.append(A)
        Z = self.network_[self.n_layers - 1] @ A
        A = expit(Z)
        
        outputs.append(Z)
        result.append(A)

        return result, outputs

    
    def _random_weights(self, phi, dim1, dim2):
        if phi != 'glorot_final':
            bound = np.sqrt(6 / (dim1 + dim2 + 1))
            w_mat = np.random.uniform(-bound, bound, (dim1, dim2+1))
        else:
            bound = np.sqrt(0.5 / (dim1 + dim2 + 1))
            w_mat = np.random.uniform(-bound, bound, (dim1, dim2+1))
        return w_mat
    
    
    
    def _initialize_weights(self):
        network = []        
        if self.n_layers == 1:
            network.append(self._random_weights(self.phi, self.classes, 
                                                self.features))            
        else:
            network.append(self._random_weights(self.phi, self.n_hidden,
                                                self.features))            
            for i in range(1, self.n_layers-1):
                network.append(self._random_weights(self.phi, self.n_hidden,
                                                   self.n_hidden))                
            network.append(self._random_weights('glorot_final', self.classes,
                                                self.n_hidden))
            
        self.network_ = network
        self.grads_ = network.copy()
    

    
    def _cost(self, result_final, y):
        if self.cost == 'mse':
            return np.mean((y-result_final)**2)
        elif self.cost == 'cross-entropy':
            return -np.mean(y * np.log(result_final) + (1-y) * np.log(1 - result_final))
        else:
            raise ValueError('Check which cost function you are using.')

    def fit(self, X, y,print_progress=0):
        Xcopy = X.copy()
        Ycopy = self._encode_labels(y.copy())

        self.features = X.shape[1]
        self.classes = Ycopy.shape[0]       
        self._initialize_weights()

        self.cost_log = []   
        self.grad_log = []
        for i in range(self.n_layers):
            self.grad_log.append([])
    

        for i in range(self.epochs):
            result, outputs = self._feedforward(Xcopy)
            self._get_gradient(result, outputs, Ycopy)
            if print_progress>0 and (i+1)%print_progress==0:
                sys.stderr.write('\rEpoch: %d/%d' % (i+1, self.epochs))
                sys.stderr.flush()
            for i in range(self.n_layers):
                self.grad_log[i].append(abs(np.mean(self.grads_[i])))
  
          
            self.cost_log.append(self._cost(result[-1], Ycopy))
           
                
            for j in range(self.n_layers): 
                self.network_[j] -= self.eta* self.grads_[j]
          
            
    
    def predict(self, X):
        result, _ = self._feedforward(X)
        yhat = np.argmax(result[-1], axis=0)
        return yhat
    
    
    
    # these are just here to make the estimator compliant with sklearn's API
    def get_params(self, deep=True):
        return {
            'eta': self.eta, 'C': self.l2_C, 'phi': self.phi,
            'cost': self.cost, 'nl_hidden': self.nl_hidden,
            'n_hidden': self.n_hidden, 'epochs': self.epochs
        }
    
    def set_params(self, **parameters):
        for parameter, value in parameters.items():
            setattr(self, parameter, value)
        return self
